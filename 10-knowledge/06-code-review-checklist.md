# Code Review 检查清单

## 功能正确性
- [ ] 代码实现了需求文档中的功能
- [ ] 边界条件已处理
- [ ] 异常流程已处理
- [ ] 没有明显的逻辑错误

## 代码质量
- [ ] 命名清晰、有意义
- [ ] 函数/方法长度合理（<50行）
- [ ] 圈复杂度合理（<10）
- [ ] 没有代码重复（DRY原则）
- [ ] 注释清晰、必要

## 安全性
- [ ] 没有 SQL 注入风险
- [ ] 没有 XSS 风险
- [ ] 敏感信息未硬编码
- [ ] 权限校验已添加
- [ ] 输入已校验

## 性能
- [ ] 没有 N+1 查询
- [ ] 大数据量场景已考虑
- [ ] 缓存使用合理
- [ ] 没有内存泄漏风险

## 测试
- [ ] 单元测试已添加
- [ ] 测试覆盖率达标（>80%）
- [ ] 关键路径有集成测试

## 文档
- [ ] 接口文档已更新
- [ ] 复杂逻辑有注释说明
- [ ] 变更已记录在 CHANGELOG
