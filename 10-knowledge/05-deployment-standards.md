# 部署规范

> 配置与密钥分离；密钥不入库。发布必须可验证、可回滚。

## 硬规则

1. **健康检查**：HTTP `GET /actuator/health`（或等价）；编排探针指向该路径，失败不得切流量
2. **镜像标签**：生产镜像 tag **锁定 commit SHA**（或 `app:<git-sha>`），禁止仅用 `latest` 上生产
3. **滚动发布**：`maxUnavailable=0`（先起新再停旧）；无法满足时须书面豁免
4. **端口约定**（默认）：
   - 应用 HTTP：`8080`
   - 管理/指标：`9090`（不对外网裸露）
5. **配置**：环境变量 / 密钥注入；仓库只保留 `*.env.example`
6. **迁移**：发布前评估；破坏性迁移与应用版本绑定，失败可回滚

## 环境

| 环境 | 用途 | 数据 |
|------|------|------|
| dev | 本地/开发 | 可销毁 |
| test | 联调/验收 | 脱敏或造数 |
| prod | 生产 | 严禁用开发密钥 |

## 发布检查

- [ ] 依赖服务就绪
- [ ] 迁移脚本已执行/已评估
- [ ] 健康检查通过
- [ ] 回滚步骤已写明（上一镜像 SHA + 配置版本）
- [ ] 密钥未写入仓库与镜像层
