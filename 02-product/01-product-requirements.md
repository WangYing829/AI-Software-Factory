## 2. 项目概述

### 2.1 背景
运维与业务值班依赖群聊与表格跟踪告警，缺少统一认领与关闭闭环，导致重复处理、责任不清、复盘困难。DemoAlert 提供轻量告警值班台，假设告警来自外部推送或手工录入。

### 2.2 目标
- **一句话**：把告警从「收到」推进到「关闭」，并留下可查询、可导出的记录。
- **可衡量**：P0 告警认领中位时长可统计；关闭记录原因完整率 100%；管理员可查看未关闭清单。

### 2.3 范围

| In Scope | Out of Scope |
|----------|--------------|
| 登录与 RBAC | 采集探针 / Agent |
| 告警列表/详情/认领/关闭 | 智能根因、移动 App |
| 组织数据隔离 | 多租户计费 |
| CSV 导出（P1） | 复杂工作流引擎 |

## 3. 用户画像

### 3.1 用户角色列表

| 角色 | 标识 | 诉求 | 关键场景 |
|------|------|------|----------|
| 系统管理员 | admin | 账号与权限可控 | 开通用户、排查权限 |
| 值班操作员 | operator | 快速认领并关闭 | 值班处置告警 |
| 只读观察者 | viewer | 了解进展 | 查看列表与详情 |

### 3.2 用户场景
1. 值班开始：operator 打开列表，筛选 `open`，认领后处理。
2. 业务旁观：viewer 查看某告警是否已关闭。
3. 管理：admin 停用离职账号，查看跨组织积压（演示简化）。

## 4. 功能需求

### 4.1 功能模块划分

| 模块 | 说明 | 优先级 |
|------|------|--------|
| M1 认证与账号 | 登录；用户管理（P1） | P0/P1 |
| M2 告警值班 | 列表/详情/认领/关闭/手工创建/导出 | P0/P1 |

### 4.2 功能详细描述

#### F-001 登录
- **描述**: 用户名密码登录并签发 Token
- **角色**: 全角色
- **前置**: 账号 active；**后置**: 持有有效 Token
- **正常流程**: 打开登录页 → 提交凭证 → 跳转原目标页
- **异常流程**: 停用账号 → 提示失败且不签发 Token
- **业务规则**: 敏感信息不写日志明文
- **优先级**: P0

#### F-002 告警列表与筛选
- **描述**: 分页查询告警，支持状态与关键字
- **角色**: admin / operator / viewer
- **前置**: 已登录
- **正常流程**: 进入列表 → 设筛选 → 查看分页结果
- **异常流程**: 无权限组织数据 → 不返回他组织行
- **业务规则**: 非 admin 强制本组织过滤
- **优先级**: P0

#### F-003 告警详情
- **描述**: 展示告警字段与事件时间线
- **角色**: admin / operator / viewer
- **正常流程**: 自列表进入详情 → 只读查看
- **异常流程**: ID 不存在或越权 → 404/403
- **优先级**: P0

#### F-004 认领告警
- **描述**: `open` → `acked`，绑定处理人
- **角色**: admin / operator
- **前置**: 告警状态为 open
- **后置**: 状态 acked，记录认领时间与人
- **正常流程**: 详情页点认领 → 成功刷新状态
- **异常流程**: 已被他人认领 → 提示当前处理人
- **业务规则**: viewer 无此操作
- **优先级**: P0

#### F-005 关闭告警
- **描述**: `acked` → `closed`，必填原因分类
- **角色**: admin / operator
- **前置**: 状态为 acked（admin 强制关闭 open 为 P1）
- **后置**: 状态 closed，写入关闭事件
- **正常流程**: 选择原因 → 确认关闭 → 列表可见 closed
- **异常流程**: 原因为空 → 校验失败；viewer → 403
- **业务规则**: 原因 ∈ {误报, 已修复, 转交, 其他}
- **优先级**: P0

#### F-006 手工创建告警（P1）
- **描述**: 无外部推送时录入标题、级别、组织等
- **角色**: admin / operator
- **优先级**: P1

#### F-007 导出 CSV（P1）
- **描述**: 导出当前筛选结果，上限示例 5000 行
- **角色**: admin / operator
- **优先级**: P1

## 5. 非功能需求

### 5.1 性能需求
- 列表查询 P95 &lt; 500ms（演示环境目标，非正式 SLA）

### 5.2 安全需求
- 默认拒绝；RBAC + 组织数据范围；管理端口不公网暴露

### 5.3 兼容性需求
- 现代浏览器最近两个大版本；API `/api/v1`

## 6. 界面需求

### 6.1 页面清单
对齐 `01-business/06-page-design.md`：登录、工作台、业务列表/详情/编辑、报表（P1）、系统管理（P1）。

### 6.2 关键页面
- **告警列表**：筛选 + 表格 + 认领入口；viewer 无写按钮
- **告警详情**：字段区 + 时间线 + 认领/关闭操作区

## 7. 数据需求

### 7.1 数据实体
对齐 `01-business/07-data-dictionary.md`：User / Role / Org / BizEntity(告警) / BizEvent。

### 7.2 数据流转
外部/手工 → 告警(open) → 认领事件 → 关闭事件 → 导出文件。
