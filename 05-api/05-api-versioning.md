# API 版本策略

## 版本规则
- URL 前缀：`/api/v{major}`
- **破坏性变更** → 升 major；兼容新增 → 同版本内演进
- OpenAPI `info.version` 与产品版本可分离，但发版说明需对照

## 兼容性承诺
| 变更类型 | 是否破坏 | 处理 |
|----------|----------|------|
| 新增字段/接口 | 否 | 文档更新 |
| 删字段/改语义 | 是 | 新 major 或弃用期 |
| 改错误码语义 | 视情况 | 公告 |

## 弃用流程
1. 文档标记 Deprecated + 截止日期  
2. 响应头或文档提示继任版本  
3. 到期移除并记入 CHANGELOG
