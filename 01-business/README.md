# 01 - Business（业务梳理）

## 当前阶段目标
将业务需求转化为结构化的产品输入。

## 文件说明

| 文件 | 内容 | AI 协作方式 |
|------|------|-------------|
| `01-project-background.md` | 项目背景、目标、范围 | 你编写，AI 补充完善 |
| `02-user-roles.md` | 用户角色与权限矩阵 | 你定义，AI 扩展细节 |
| `03-business-flow.md` | 核心业务流程图/描述 | 你梳理，AI 优化表达 |
| `04-functional-list.md` | 功能清单（模块→功能→子功能） | 你列出，AI 补全遗漏 |
| `05-permission-design.md` | 权限设计（RBAC/ABAC） | AI 根据角色生成 |
| `06-page-design.md` | 页面清单与跳转关系 | AI 根据功能生成 |
| `07-data-dictionary.md` | 核心数据实体与字段 | AI 根据业务推导 |
| `08-prototype/` | 原型图/线框图 | 你画草图，AI 生成详细原型 |

## 操作建议

1. 先用 Markdown 手写核心内容（不要追求完美）
2. 将写好的文档作为 Context，让 AI 帮你：
   - 检查遗漏
   - 补充细节
   - 生成数据字典
   - 推导页面结构
3. 每完成一个文件，在 `11-context/` 中更新进度

## 输出标准

完成本阶段后，你应该能回答：
- 这个系统有哪些用户角色？
- 每个角色的核心业务流程是什么？
- 系统包含哪些功能模块？
- 每个模块有哪些具体功能？
- 功能之间的数据流转关系？
