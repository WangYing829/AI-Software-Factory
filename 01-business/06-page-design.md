# 页面设计

> 模板说明：页面由功能清单推导；`[示例: …]` 替换为真实业务名。线框放 `08-prototype/`。

## 1. 页面清单

| 页面 ID | 页面名称 | 路由 | 主要角色 | 关联功能 | 优先级 |
|---------|----------|------|----------|----------|--------|
| P-LOGIN | 登录页 | `/login` | 全员 | 认证 | P0 |
| P-HOME | 工作台 | `/` | 全员 | 概览 | P0 |
| P-LIST | `[示例: 业务列表]` | `/biz` | operator, viewer | 查询 | P0 |
| P-DETAIL | `[示例: 业务详情]` | `/biz/:id` | operator, viewer | 详情 | P0 |
| P-FORM | `[示例: 业务编辑]` | `/biz/:id/edit` | operator | 新建/编辑 | P0 |
| P-RPT | `[示例: 报表]` | `/reports` | viewer+ | 报表 | P1 |
| P-ADMIN | 系统管理 | `/admin` | admin | 配置 | P1 |

## 2. 跳转关系（文字版）

```
登录 → 工作台
工作台 → 业务列表 / 报表 / 系统管理
业务列表 → 详情 → 编辑 → 返回列表
未登录访问受保护路由 → 登录（带回跳）
无权限 → 403 / 无权限提示页
```

## 3. 关键页面说明（骨架）

### P-LIST `[示例: 业务列表]`
- **目的**: 检索与进入详情
- **区块**: 筛选区 / 表格 / 分页 / 操作列
- **主要操作**: 查询、新建、导出、进入详情
- **权限**: `biz:entity:read`；写操作需 `biz:entity:write`

### P-DETAIL `[示例: 业务详情]`
- **目的**: 只读展示 + 进入编辑
- **区块**: 基本信息 / 状态时间线 / 关联数据
- **主要操作**: 编辑、返回

## 4. 全局交互约定

| 项 | 约定 |
|----|------|
| 空状态 | 明确文案 + 可选 CTA（新建 / 清空筛选） |
| 加载 | 列表骨架屏或表格 loading |
| 错误 | 可重试；鉴权失败跳登录 |
| 面包屑 | 工作台 › 模块 › 当前页 |

## 5. 验收检查

- [ ] 每个 P0 功能至少映射到一个页面或明确为非页面能力（如定时任务）
- [ ] 跳转闭环：列表 ↔ 详情 ↔ 编辑
- [ ] 权限不足路径有页面表现
