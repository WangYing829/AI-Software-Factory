# 功能清单

> 演示虚构：DemoAlert。

## 模块1：认证与账号

### 功能1.1：登录
- **功能描述**: 用户名密码登录，签发访问令牌
- **优先级**: P0
- **关联角色**: admin, operator, viewer
- **前置条件**: 账号 active
- **后置条件**: 获得会话/Token
- **输入**: username, password
- **输出**: Token / 用户信息
- **业务规则**: 停用账号不可登录；错误提示不区分「用户不存在/密码错」的细节到可枚举程度（演示可简化）

### 功能1.2：用户管理（最小）
- **功能描述**: admin 启用/停用用户、分配角色
- **优先级**: P1
- **关联角色**: admin
- **业务规则**: 不可删除最后一个 admin（演示规则）

## 模块2：告警值班

### 功能2.1：告警列表与筛选
- **功能描述**: 分页列表，按状态/关键字筛选
- **优先级**: P0
- **关联角色**: admin, operator, viewer
- **输入**: page, size, status, keyword
- **输出**: 列表 + total
- **业务规则**: 非 admin 仅本组织

### 功能2.2：告警详情
- **功能描述**: 查看告警字段与事件时间线
- **优先级**: P0
- **关联角色**: admin, operator, viewer

### 功能2.3：认领告警
- **功能描述**: 将 open 告警变为 acked 并绑定处理人
- **优先级**: P0
- **关联角色**: admin, operator
- **业务规则**: 已被认领则失败

### 功能2.4：关闭告警
- **功能描述**: acked → closed，必填原因分类
- **优先级**: P0
- **关联角色**: admin, operator
- **业务规则**: 原因枚举见流程文档

### 功能2.5：手工创建告警
- **功能描述**: 无外部推送时手工录入
- **优先级**: P1
- **关联角色**: admin, operator

### 功能2.6：导出告警 CSV
- **功能描述**: 导出当前筛选结果
- **优先级**: P1
- **关联角色**: admin, operator
- **业务规则**: 最大行数限制（示例 5000）

---

## 功能优先级汇总

| 优先级 | 功能数量 | 功能列表 |
|--------|----------|----------|
| P0（必须有） | 5 | 登录、列表、详情、认领、关闭 |
| P1（应该有） | 3 | 用户管理、手工创建、导出 |
| P2（可以有） | 0 | （转派/批量关闭等后续） |
