# 权限设计（RBAC）

> 模板说明：Fork / 复用本工厂时，将 `[示例: …]` 替换为真实业务；本文件骨架需保留，内容由角色与功能清单推导。

## 1. 模型选择

| 项 | 取值 |
|----|------|
| 模型 | RBAC（角色-权限）；必要时扩展资源维度 ABAC |
| 默认拒绝 | 未显式授权一律拒绝 |
| 数据范围 | 按角色配置：全部 / 本组织 / 本人 / 自定义 |

## 2. 角色清单（与 `02-user-roles.md` 对齐）

| 角色编码 | 角色名称 | 说明 | 示例 |
|----------|----------|------|------|
| `admin` | 系统管理员 | 配置与账号 | `[示例: 平台超管]` |
| `operator` | 业务操作员 | 日常业务操作 | `[示例: 监控值班员]` |
| `viewer` | 只读用户 | 查询与报表 | `[示例: 业务观察者]` |

## 3. RBAC 矩阵（最小骨架）

图例：`C` 创建 / `R` 读 / `U` 改 / `D` 删 / `X` 执行 / `-` 无

| 资源 / 能力 | admin | operator | viewer |
|-------------|-------|----------|--------|
| `[示例: 用户管理]` | CRUD | - | - |
| `[示例: 业务主数据]` | CRUD | CRU | R |
| `[示例: 业务单据]` | CRUD | CRUX | R |
| `[示例: 报表导出]` | X | X | R |
| 系统配置 | CRUD | - | - |

## 4. 权限点编码约定

```
<域>:<资源>:<动作>
示例: biz:order:create / sys:user:delete / rpt:dashboard:export
```

| 权限点 | 说明 | 默认角色 |
|--------|------|----------|
| `sys:user:manage` | 用户与角色管理 | admin |
| `biz:entity:read` | 业务实体只读 | admin, operator, viewer |
| `biz:entity:write` | 业务实体写 | admin, operator |
| `rpt:export` | 导出 | admin, operator |

## 5. 数据范围规则

| 角色 | 默认数据范围 | 备注 |
|------|--------------|------|
| admin | 全部 | — |
| operator | 本组织 | 可按项目覆盖 |
| viewer | 本组织只读 | 禁止写接口 |
